Version 5.2

  • Huber Circulating chiller support.
  • List of recent js files
  • Tabs in JS editor
  • Button "Connect All"
  • Settings Dialog

Version 5.1

  • JavaScript engine is updated up to version v3.0.0. Now await and async instructions are available in JS.
  • Logger starts a new file when the file size (or the nubmer of lines) is more than a user defined value.
  • The status bar (with log messages) keeps a limited user defined number of messages.

Version 4.2

  • Event hadlers MeasurementStarted, MeasurementEnded are assigned when palmsense.PSCommSimpleWPF is set.

Version 4.1

  • MethodScript functionality was added for EmStat Pico. Use palmsense.RunMethodScript(string path, string csv="") to execute MethodScript and save it to a csv file.
  • PalmSens SDK for WPF version updated from 5.7 to 5.9.

Version 4.0

  • The new chamber supports BME680 temperature/humidity sensor. Check help page to know how to switch between different modes of operation.
  • Number of points on the temperature chart is increased up to 100.

Version 3.2

  • Command palmsense.SaveMeasurements is now a sync method. That means that JS execution will be halted untill CSV file is saved.

Version 3.1

  • Jint interpreter updated to v3.0.0-beta-2036
  • JavaScript execution can be stopped when the current statement is complete with Stop button. JavaScript execution can be immediately terminated with Kill button.
  • JavaScript can be saved to hard disk. Button "Save" added to Main menu.
  • Added support for multichannel Palmsense measurements. Execute palmsense.Measure(channel) to measure a single channel. Ommit argument channel for measurements without MUX.
    • Alternatively: If palmsense.Measure(ch) doesn't work or sequentiall measurements needed, try palmsense.SetupMUX(bool[] array), where array is an array of booleans representing the channel states, for example {true, true, false, false, false, false, false} to measure first two channels.
  • Temperature regulator set value should be given in 0.1 Celcius deg. Important! setTemp(37) will try to run PID at 3.7C which is almost unreachable for a small Peltier and completely impossible in a setup with only heating element.

Version 3.0

  • The new chamber supports heating element (not Peltier). Temperature can be set using menu button 'Set temp' or in Java Script using arduino.setTemp(int t).
  • Blinking indicator showing communication to Arduino.

Version 2.4

  • Support of AHT10 temperature/humidity sensor board. Use AHT10 compiler tag to build an application with AHT10 support. Humidity can be read from JS using arduino.Humidity.

Version 2.3

  • Max flow rate acceptable by arduino.setFlow is back to 200 ccm/min.

Version 2.2

  • Bug with arduino.setFlow fixed.

Version 2.1

  • Read arduino.Temperature to get current temperature.

Version 2.0

  • Bug with restore of the width of JavaScript list has been fixed.
  • This version is designed for the new arduino shield. The new shield has external DAC and ADC modules. If you need to work with the old shield, remove compilation symbol I2C from project build settings. External DAC can output voltage from 0 to 5V with 12 bit resolution. External ADC can measure +/- 6.144V with 16 bit resolution, thus 1 bit = 0.1875mV. The new shield works only with the new firmware of arduino.
  • Event handler SimpleCurveStartReceivingData removed after Palmsense measurement completion. It can solve the bug with doubling of data in CSV file.

Version 1.4

  • JSListWidth, last selected COM port are now preserved between sessions.
  • Bug with NullReferenceException during execution of Measure method has been fixed.

Version 1.3

  • If Don't show next time checkbox is set, What's New dialog won't be displayed next time.

Version 1.2

  • Function SaveMeasurements of IPalmsense interface now has the 2nd optional parameter csv which is defualt to False. If csv is True Palmsense measurements are saved in CSV table rather than pssession file.
    Example:
    palmsense.SaveMeasurements("test.csv", true);
